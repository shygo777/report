🟩 選擇排序法

Selection Sort - 深入理解與可視化

主頁 氣泡排序 插入排序

📚 原理說明

選擇排序（Selection Sort）是一種簡單的排序算法，它通過在未排序的部分中反覆選擇最小（或最大）元素，並將其放在已排序部分的末尾。


算法名稱來源：每次都「選擇」（select）未排序部分中最小的元素。


核心概念：

  • 將陣列分為已排序和未排序兩部分
  • 在未排序部分中尋找最小值
  • 將最小值移動到已排序部分末尾
  • 重複此過程直到整個陣列排序完成

⚙️ 排序過程

演算法步驟：

  1. 初始化：設 i 從 0 開始，設定 minIndex = i
  2. 尋找最小值：從 i+1 開始遍歷到陣列末尾
  3. 比較更新：若其他元素更小，更新 minIndex
  4. 交換元素：將位置 i 與 minIndex 的元素交換
  5. 移進指標：i 遞增，重複步驟 2-4
  6. 完成：當 i 遍歷到倒數第二個元素時排序完成

圖示示例：

初始陣列： [5, 2, 8, 1, 9]

逐步過程：

輪次 已排序 未排序 最小值 動作
初始 [ ] [5, 2, 8, 1, 9] - -
第1輪 [1] [5, 2, 8, 9] 1 1和5交換
第2輪 [1, 2] [5, 8, 9] 2 2和5交換
第3輪 [1, 2, 5] [8, 9] 5 5已在位
第4輪 [1, 2, 5, 8] [9] 8 已完成
結果 [1, 2, 5, 8, 9] [ ] - ✓ 排序完成

📊 時間與空間複雜度

時間複雜度

最好情況： O(n²)

（即使已排序，仍需全部掃描）


平均情況： O(n²)

（隨機陣列）


最壞情況： O(n²)

（逆序陣列）

空間複雜度

輔助空間： O(1)

（只需常數額外空間）


原地排序： ✓ 是

（在原陣列上進行排序）


穩定性： ✗ 不穩定

（相等元素相對位置可能改變）

性能評價

優點：

  • ✓ 實現簡單
  • ✓ 原地排序
  • ✓ 交換次數少

缺點：

  • ✗ 效率低（O(n²)）
  • ✗ 不穩定
  • ✗ 不適合大數據

💻 代碼實現

基礎版本：

function selectionSort(arr) { const n = arr.length; // 外層迴圈：控制已排序部分長度 for (let i = 0; i < n - 1; i++) { let minIndex = i; // 內層迴圈：在未排序部分找最小值 for (let j = i + 1; j < n; j++) { if (arr[j] < arr[minIndex]) { minIndex = j; } } // 交換最小值到正確位置 if (minIndex !== i) { let temp = arr[i]; arr[i] = arr[minIndex]; arr[minIndex] = temp; } } return arr; }

使用示例：

// 使用示例 let numbers = [64, 34, 25, 12, 22, 11, 90]; console.log("原始陣列:", numbers); let sorted = selectionSort([...numbers]); console.log("排序後:", sorted); // 輸出: [11, 12, 22, 25, 34, 64, 90] // 追蹤交換過程 function selectionSortWithTrace(arr) { const n = arr.length; let swapCount = 0; for (let i = 0; i < n - 1; i++) { let minIndex = i; for (let j = i + 1; j < n; j++) { if (arr[j] < arr[minIndex]) { minIndex = j; } } if (minIndex !== i) { console.log(`交換位置 ${i} 和 ${minIndex}: ${arr[i]} ↔ ${arr[minIndex]}`); let temp = arr[i]; arr[i] = arr[minIndex]; arr[minIndex] = temp; swapCount++; } } console.log(`總交換次數: ${swapCount}`); return arr; }

🔍 應用場景與對比

何時使用選擇排序：

  • ✓ 教育用途 - 學習排序算法基礎
  • ✓ 交換操作昂貴的場景（交換次數最多 n-1 次）
  • ✓ 小規模數據 - 數據量在 10-100 個元素左右
  • ✓ 原地排序且空間有限的場景
