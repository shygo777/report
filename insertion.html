📍 插入排序法

Insertion Sort - 深入理解與可視化

主頁 氣泡排序 選擇排序

📚 原理說明

插入排序法（Insertion Sort）是一種簡單直觀的排序算法，它的工作原理類似於整理手中的撲克牌。每次拿起一張新牌時，我們會將它插入到已經排序的牌中的正確位置。


算法名稱來源：通過逐個抽出未排序區域的元素，並將其插入到已排序區域的正確位置而得名。


核心概念：

  • 將陣列分為已排序和未排序兩部分
  • 從未排序部分取出一個元素
  • 在已排序部分中找到正確位置
  • 將元素插入，其他元素向後移動
  • 重複此過程直到所有元素都被插入

⚙️ 排序過程

演算法步驟：

  1. 初始化：第一個元素默認已排序，從第二個元素開始
  2. 關鍵元素：選出未排序部分的第一個元素作為要插入的元素
  3. 比較：將該元素與已排序部分的元素逐一比較
  4. 移動：比該元素大的元素向右移動一個位置
  5. 插入：將關鍵元素放入正確位置
  6. 重複：對下一個未排序元素重復步驟2-5

數值示例：

原始陣列： [5, 2, 8, 1, 9]

詳細過程：

  • 第1步： [5 | 2, 8, 1, 9] → 已排序: [5]，取出 2
  • 2 < 5，所以 2 插到 5 前面 → [2, 5 | 8, 1, 9]
  • 第2步： [2, 5 | 8, 1, 9] → 已排序: [2, 5]，取出 8
  • 8 > 5，8 在最後 → [2, 5, 8 | 1, 9]
  • 第3步： [2, 5, 8 | 1, 9] → 已排序: [2, 5, 8]，取出 1
  • 1 < 2，1 插到最前面 → [1, 2, 5, 8 | 9]
  • 第4步： [1, 2, 5, 8 | 9] → 已排序: [1, 2, 5, 8]，取出 9
  • 9 > 8，9 在最後 → [1, 2, 5, 8, 9]

結果：所有元素已正確排序 [1, 2, 5, 8, 9]

📊 時間與空間複雜度

時間複雜度

最好情況： O(n)

（已排序陣列，無移動）


平均情況： O(n²)

（隨機陣列）


最壞情況： O(n²)

（逆序陣列）

空間複雜度

輔助空間： O(1)

（只需常數額外空間）


原地排序： ✓ 是

（在原陣列上進行排序）


穩定性： ✓ 穩定

（相等元素相對位置不變）

性能評價

優點：

  • ✓ 實現簡單
  • ✓ 原地排序
  • ✓ 穩定排序
  • ✓ 對已排序數據高效

缺點：

  • ✗ 效率低（O(n²)）
  • ✗ 需要多次移動
  • ✗ 不適合大規模數據

💻 代碼實現

基礎版本：

function insertionSort(arr) { const n = arr.length; // 從第二個元素開始 for (let i = 1; i < n; i++) { const key = arr[i]; // 要插入的元素 let j = i - 1; // 已排序部分的索引 // 在已排序部分中找到正確位置 while (j >= 0 && arr[j] > key) { arr[j + 1] = arr[j]; // 向右移動 j--; } // 插入元素到正確位置 arr[j + 1] = key; } return arr; }

帶追蹤的版本：

function insertionSortWithTrace(arr) { const n = arr.length; let comparisons = 0; let shifts = 0; for (let i = 1; i < n; i++) { const key = arr[i]; let j = i - 1; console.log(`插入第 ${i} 個元素: ${key}`); while (j >= 0 && arr[j] > key) { comparisons++; arr[j + 1] = arr[j]; shifts++; j--; } arr[j + 1] = key; console.log(`結果: [${arr.join(', ')}]`); } console.log(`總比較次數: ${comparisons}`); console.log(`總移動次數: ${shifts}`); return arr; }

使用示例：

// 基本用法 let numbers = [64, 34, 25, 12, 22, 11, 90]; console.log("原始陣列:", numbers); let sorted = insertionSort([...numbers]); console.log("排序後:", sorted); // 輸出: [11, 12, 22, 25, 34, 64, 90] // 帶詳細追蹤 console.log("\n=== 詳細過程 ==="); insertionSortWithTrace([5, 2, 8, 1, 9]);

🔍 應用場景與對比

何時使用插入排序：

  • ✓ 小規模數據 - 數據量在 50-1000 個元素左右
  • ✓ 幾乎有序的數據 - 最優時間複雜度為 O(n)
  • ✓ 線上排序 - 實時接收數據並排序
  • ✓ 需要穩定排序的場景
  • ✓ 外部排序或與其他算法混合使用

與其他排序算法對比：

算法 平均時間複雜度 最壞情況 空間複雜度 穩定性
氣泡排序 O(n²) O(n²) O(1) 穩定
選擇排序 O(n²) O(n²) O(1) 不穩定
插入排序 O(n²) O(n²) O(1) 穩定
快速排序 O(n log n) O(n²) O(log n) 不穩定
合併排序 O(n log n) O(n log n) O(n) 穩定
