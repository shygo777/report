# 排序算法報告

https://shygo777.github.io/report/

## 前言
本報告以五種常見排序法為主題，結合理論說明與程式實驗，探討排序算法在不同資料規模下的效能變化。排序算法是資料結構與演算法中的基礎問題，瞭解它們的操作方式與時間、空間複雜度，有助於選擇適合的排序策略。

## 排序法原理

### 1. 氣泡排序（Bubble Sort）
- 基本原理：重複掃描陣列，對相鄰元素兩兩比較，若順序錯誤則交換。每一輪會將最大（或最小）元素「冒泡」到末端。
- 操作方式：從頭到尾依序比較相鄰元素，交換後繼續比較；完成一輪後，最右邊元素為已排序項，下一輪可縮小比較範圍。

### 2. 選擇排序（Selection Sort）
- 基本原理：在未排序區間中選出最小元素，然後與未排序區間的第一個元素交換，使其成為已排序區間的新增元素。
- 操作方式：每一輪從剩餘元素中找最小值，將其放到當前位置；以此逐步擴大已排序區間。

### 3. 插入排序（Insertion Sort）
- 基本原理：將資料分成已排序區間和未排序區間，從未排序區間取出元素，插入到已排序區間的正確位置。
- 操作方式：從第二個元素開始，將當前元素與左側已排序元素比較，移動較大元素並插入正確位置，類似整理手牌。

### 4. 合併排序（Merge Sort）
- 基本原理：分治策略。將陣列遞迴分成兩半，分別排序後再合併成單一已排序陣列。
- 操作方式：遞迴拆分直到單一元素，然後比較兩個子陣列的頭部元素，依序合併成有序陣列。

### 5. 堆積排序（Heap Sort）
- 基本原理：先將資料建構成最大堆，再不斷取出堆頂元素並重建堆，最後得到排序結果。
- 操作方式：使用二元堆結構維持堆性質，逐一將最大值與末端交換，縮小堆範圍並重新 heapify。

## 複雜度分析

| 排序法 | 最佳時間 | 平均時間 | 最差時間 | 空間複雜度 | 穩定性 |
|---|---|---|---|---|---|
| 氣泡排序 | O(n^2) | O(n^2) | O(n^2) | O(1) | 穩定 |
| 選擇排序 | O(n^2) | O(n^2) | O(n^2) | O(1) | 不穩定 |
| 插入排序 | O(n) | O(n^2) | O(n^2) | O(1) | 穩定 |
| 合併排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | 穩定 |
| 堆積排序 | O(n log n) | O(n log n) | O(n log n) | O(1) * | 不穩定 |

> 
> *實作上若使用標準 `heapq` 或額外陣列，會消耗 O(n) 額外空間；純原地堆積排序理論上可達 O(1) 額外空間。

## 實驗設計

本報告以 `benchmark_sort.py` 程式實作五種排序法，測量不同資料規模下的平均執行時間。實驗設定如下：

- 資料型態：隨機整數陣列
- 資料規模：n = 100、500、1000、2000
- 每個演算法在相同資料上執行 3 次，取平均時間
- 評估環境：Python 3.13、單次執行時間測量

## 實驗結果

| n | Bubble Sort (秒) | Selection Sort (秒) | Insertion Sort (秒) | Merge Sort (秒) | Heap Sort (秒) |
|---|---|---|---|---|---|
| 100 | 0.000366 | 0.000168 | 0.000186 | 0.000137 | 0.000022 |
| 500 | 0.010064 | 0.004984 | 0.004928 | 0.000851 | 0.000109 |
| 1000 | 0.045112 | 0.021851 | 0.020669 | 0.001859 | 0.000228 |
| 2000 | 0.188663 | 0.084602 | 0.087137 | 0.004379 | 0.000516 |

### 效能比較

- 資料規模越大，O(n^2) 演算法（氣泡、選擇、插入）的時間成本顯著上升。
- 合併排序與堆積排序皆表現出近似 O(n log n) 的成長，當 n = 2000 時執行時間仍遠低於 O(n^2) 演算法。
- 在本實驗中，堆積排序最小、合併排序次之，插入與選擇排序類似，氣泡排序最慢。

## 結果分析

- **氣泡排序**：最直觀，但在大資料時效能最差，適合教學或已近排序的資料。
- **選擇排序**：讀寫次數較少，仍屬 O(n^2)，對大資料不適合。
- **插入排序**：對近乎排序資料適用，可降為 O(n)，但平均情況仍為 O(n^2)。
- **合併排序**：穩定且時間複雜度穩定，適合大資料量，但需要額外空間。
- **堆積排序**：理論上可原地排序且穩定的 O(n log n)，實驗中的 `heapq` 版本也展現極佳效能。

## 心得與結論

本次實驗結果與理論分析一致：

- O(n^2) 演算法在 n 增加時速度急劇下降，顯示平方級別不適合大型資料。
- O(n log n) 演算法在 n 增長時仍維持相對平緩的時間成本，適合較大資料集。
- 若材料量較小或資料已近排序，可考慮插入排序；若需通用且穩定的解法，首選合併排序；若需節省額外空間，堆積排序為較好選擇。

### 建議

1. 小資料、教學示範：使用氣泡、選擇、插入排序理解基礎概念。
2. 大資料排序：優先考慮合併排序或堆積排序。
3. 若需要穩定排序且額外空間不是問題：合併排序效果最佳。
4. 若實作原地排序且空間敏感：堆積排序更適合。
